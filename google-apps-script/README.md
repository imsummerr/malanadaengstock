# 🔔 ระบบแจ้งเตือน LINE — ของเข้า + ของที่ต้องทิ้ง

มี 2 การแจ้งเตือนอัตโนมัติ:

| | แจ้งเมื่อไหร่ | แจ้งอะไร |
|---|---|---|
| 📦 **ของเข้าใหม่** | ทันทีที่กรอกของเข้า (ภายใน ~5 นาที) | ของทุกรายการที่เข้า + วันที่ควรทิ้งของแต่ละรายการ |
| 🗑️ **ของครบกำหนดทิ้ง** | ทุกวันตอน **1 ทุ่ม (19:00 น.)** | รายการไหนต้องเอาออก/ทิ้ง "วันนี้" (รวมของที่เลยกำหนดแล้ว) |

วันที่ต้องทิ้งคำนวณจากอายุของแต่ละรายการ (นับรวมวันที่ของเข้า):

> เช่น **สันคอสไลด์ อยู่ได้ 5 วัน** — เข้าวันที่ 1/7/2026 → ต้องเคลียร์ทิ้งวันที่ 5/7/2026

ตัวอย่างข้อความแจ้งของเข้าใหม่:

```
📦 ของเข้าใหม่ (1/7/2026 14:32 น.)

📍 แบริ่ง — ผู้ตรวจ: สมชาย
• สันคอสไลด์ 3 ถุง → ทิ้ง 5/7/2026
• หมึก 2 ถุง → ทิ้ง 10/7/2026
• ปูอัด 1 ถุง → ทิ้ง 14/7/2026
```

### อายุของแต่ละรายการ

| อายุ | รายการ |
|---|---|
| **5 วัน** | สันคอสไลด์, สามชั้นสไลด์, เนื้อแดง, กุ้ง |
| **10 วัน** | หมึก, ปลาดอลลี่, ปลาหมึกกรอบ, แมงกะพรุน |
| **14 วัน** | ปูอัด, เต้าหู้ชีส, ชีสหลายสี, กุ้งพันสาหร่าย, ไส้กรอกพันเบคอน, ฟองเต้าหู้สามเหลี่ยม, ไส้กรอกหนังกรอบ, ไส้กรอกชมพู |
| **30 วัน (1 เดือน)** | รากบัว, ต็อก (แป้งต็อก) |
| **7 วัน (ค่าเริ่มต้น)** | รายการอื่น ๆ ที่ไม่ได้ระบุ |

แก้ไขอายุได้ที่ตาราง `ITEM_EXPIRY_DAYS` ทั้งใน `line-expiry-alert.gs` และใน `index.html` (มีตารางเดียวกัน 2 ที่ — แก้ให้ตรงกันทั้งคู่)

ตัวอย่างข้อความแจ้งของครบกำหนดทิ้ง (ส่งตอน 1 ทุ่ม แยกตามสาขา):

```
🔔 แจ้งเตือนของหมดอายุ (7/7/2026)

🗑️ ของที่ต้องทิ้ง "วันนี้":
📍 แบริ่ง
• สันคอสไลด์ 3 ถุง (เข้า 3/7/2026)
• ผักกาดขาว 500 กรัม (เข้า 1/7/2026)

⚠️ เลยกำหนดทิ้งแล้ว:
📍 ตลาดทรัพย์พัฒนา
• หมึก 2 ถุง (เลยมา 1 วัน)
```

## ขั้นตอนที่ 1: สร้าง LINE Bot (Messaging API)

> 💡 แนะนำให้**สร้างบอทใหม่แยกสำหรับระบบสต็อกโดยเฉพาะ** (เช่น ตั้งชื่อ "สต็อกหม่าล่าหน้าแดง") — การแจ้งเตือนสต็อกจะเด้งเข้า**คนละแชท**กับระบบอื่น (เช่น ออเดอร์ Lagombake) ไม่ปนกัน และโควต้าข้อความฟรี 200 ข้อความ/เดือนแยกกันคนละก้อนด้วย
>
> สร้าง Channel ใหม่ใต้ Provider เดิมได้เลย ไม่ต้องสร้าง Provider ใหม่ — แต่ **Channel access token กับ Your user ID ต้องคัดลอกจาก Channel ใหม่เท่านั้น** ห้ามเอาของบอทเก่ามาใช้ปน

> ⚠️ LINE Notify ปิดบริการไปแล้ว (มี.ค. 2025) จึงต้องใช้ **LINE Messaging API** แทน

1. เข้า https://developers.line.biz/console/ → Login ด้วยบัญชี LINE
2. สร้าง **Provider** ใหม่ (ตั้งชื่ออะไรก็ได้ เช่น "หม่าล่าหน้าแดง")
3. สร้าง **Channel** แบบ **Messaging API**
4. ไปที่แท็บ **Messaging API** → กด **Issue** ที่หัวข้อ **Channel access token (long-lived)** → คัดลอกเก็บไว้
5. ปิด **Auto-reply messages** (ในหน้า LINE Official Account Manager) เพื่อไม่ให้บอทตอบอัตโนมัติ

### เลือกปลายทาง (จะให้ส่งเข้าไหน)

**แบบง่ายสุด — Broadcast ไม่ต้องหา ID ไม่ต้องใช้ webhook (แนะนำ):**
- **ไม่ต้องตั้งค่า `LINE_TARGET_ID` เลย** — ระบบจะส่งหา "ทุกคนที่แอดบอทเป็นเพื่อน" อัตโนมัติ
- ใครอยากได้แจ้งเตือน แค่สแกน QR code ของบอท (แท็บ Messaging API) แอดเป็นเพื่อน จบ
- ⚠️ ข้อควรระวังเรื่องโควต้าฟรี 200 ข้อความ/เดือน: broadcast นับตามจำนวนผู้รับ เช่น มีคนแอดบอท 3 คน ส่ง 1 ครั้ง = นับ 3 ข้อความ — เหมาะกับทีมเล็ก (1–3 คน) ถ้าคนเยอะกว่านั้นให้ใช้แบบกลุ่มแทน (ส่งเข้ากลุ่มนับแค่ 1 ไม่ว่ากลุ่มมีกี่คน)

**แบบส่งเข้าแชทส่วนตัวของเจ้าของร้านคนเดียว:**
- ในหน้า Channel แท็บ **Basic settings** เลื่อนลงล่างสุด จะเห็น **Your user ID** (ขึ้นต้นด้วย `U...`) → ใช้ค่านี้เป็น Target ID
- อย่าลืม **แอดบอทเป็นเพื่อน** ใน LINE ก่อน (สแกน QR code ในแท็บ Messaging API)

**แบบส่งเข้ากลุ่ม (ให้พนักงานทุกคนเห็น):**

ต้องหา Group ID (ขึ้นต้นด้วย `C...`) โดยใช้ **webhook.site** ช่วยดักดู (ใช้ครั้งเดียวตอนติดตั้ง):

1. เปิด https://webhook.site → มันจะสร้าง URL เฉพาะของเราให้ (หน้าตา `https://webhook.site/xxxx-xxxx...`) → คัดลอก URL นั้น
2. ใน LINE Developers Console → Channel ของบอท → แท็บ **Messaging API** → หัวข้อ **Webhook settings** → วาง URL ลงใน **Webhook URL** → กด Update → เปิดสวิตช์ **Use webhook**
3. สร้างกลุ่ม LINE (หรือใช้กลุ่มเดิม) แล้ว**เชิญบอทเข้ากลุ่ม**
   - ถ้าเชิญไม่ได้ ให้ไปเปิดที่ LINE Official Account Manager → การตั้งค่า → เปิด "อนุญาตให้เชิญเข้ากลุ่มแชท"
4. กลับไปดูหน้า webhook.site จะเห็น request เด้งเข้ามา มองหาคำว่า `"groupId":"C..."` → คัดลอกค่า `C...` นั้น
5. เอา `C...` ไปใส่เป็น `LINE_TARGET_ID` ในขั้นตอนที่ 3
6. เสร็จแล้วกลับไป**ปิดสวิตช์ Use webhook** และลบ Webhook URL ออก (ไม่ต้องใช้อีกแล้ว — ระบบแจ้งเตือนเป็นฝ่ายส่งอย่างเดียว ไม่ต้องรับ)

## ขั้นตอนที่ 2: เพิ่มโค้ดเข้า Apps Script เดิม

1. เปิด Google Sheets ที่เก็บข้อมูลสต็อก → เมนู **ส่วนขยาย (Extensions)** → **Apps Script**
2. กดปุ่ม **+** ข้าง "ไฟล์" → **สคริปต์** → ตั้งชื่อ `line-expiry-alert`
3. คัดลอกโค้ดทั้งหมดจากไฟล์ [`line-expiry-alert.gs`](./line-expiry-alert.gs) ไปวาง → กด **บันทึก** (Ctrl+S)

---

## ขั้นตอนที่ 3: ตั้งค่า Token

1. ใน Apps Script กดไอคอน **เฟือง (Project Settings)** ด้านซ้าย
2. เลื่อนลงไปที่ **Script Properties** → กด **Add script property** เพิ่ม 2 ค่า:

| Property | Value |
|---|---|
| `LINE_CHANNEL_ACCESS_TOKEN` | token ที่คัดลอกมาจากขั้นตอนที่ 1 |
| `LINE_TARGET_ID` | *(ไม่ต้องใส่ = broadcast หาทุกคนที่แอดบอท)* หรือใส่ User ID (`U...`) / Group ID (`C...`) |

3. ในหน้า Project Settings เดียวกัน ตรวจว่า **Time zone** เป็น `(GMT+07:00) Bangkok` (ถ้าไม่ใช่ ให้แก้ เพื่อให้แจ้งเตือนตรงเวลาไทย)

## ขั้นตอนที่ 4: ทดสอบ + เปิดใช้งาน

รันฟังก์ชันจากเมนูด้านบนของ Apps Script (เลือกชื่อฟังก์ชัน → กด **Run** ครั้งแรกจะขอสิทธิ์ ให้กดอนุญาต):

1. **`testLineConnection`** — ส่งข้อความทดสอบเข้า LINE ถ้าได้รับข้อความแปลว่า token ถูกต้อง ✅
2. **`previewItems`** — ดูรายการที่จะถูกแจ้งเตือน (ดูผลใน Execution log) โดยยังไม่ส่ง LINE
3. **`testNotifyNow`** — ลองรันแจ้งเตือนของครบกำหนดจริงทันที
4. **`setupTriggers`** — รันครั้งเดียว เพื่อเปิดการแจ้งเตือนอัตโนมัติทั้ง 2 แบบ:
   - เช็คของเข้าใหม่ทุก 5 นาที → แจ้ง LINE ทันทีที่มีของเข้า
   - แจ้งของครบกำหนดทิ้งทุกวันตอนประมาณ **1 ทุ่ม (19:00–20:00 น.)**

เสร็จแล้ว! หลังจากนี้ระบบจะเช็คและแจ้งเตือนเองอัตโนมัติ 🎉

> ทดสอบแจ้งของเข้า: หลังรัน `setupTriggers` แล้ว ลองกรอก "จำนวนของเข้า" ในหน้าเว็บ 1 รายการ รอไม่เกิน 5 นาที ข้อความจะเด้งเข้า LINE (หรือรันฟังก์ชัน `checkNewIncoming` เองทันทีก็ได้)

## ปรับแต่งเพิ่มเติม

แก้ค่าด้านบนสุดของไฟล์ `line-expiry-alert.gs`:

| ตัวแปร | ค่าเริ่มต้น | ความหมาย |
|---|---|---|
| `ITEM_EXPIRY_DAYS` | ดูตารางด้านบน | อายุของแต่ละรายการ (วัน นับรวมวันของเข้า) |
| `EXPIRY_DAYS_DEFAULT` | `7` | อายุของรายการที่ไม่อยู่ในตาราง (เข้า 1/7 → ทิ้ง 7/7) |
| `NOTIFY_HOUR` | `19` | แจ้งของครบกำหนดทิ้งตอนกี่โมง (19 = 1 ทุ่ม, แก้แล้วรัน `setupTriggers` ใหม่) |
| `INCOMING_POLL_MINUTES` | `5` | เช็คของเข้าใหม่ทุกกี่นาที (ใช้ได้: 1, 5, 10, 15, 30) |
| `OVERDUE_LOOKBACK_DAYS` | `7` | แจ้งของเลยกำหนดย้อนหลังไม่เกินกี่วัน |
| `INCOMING_SHEET_NAME` | `จำนวนของเข้า` | ชื่อชีตที่เก็บข้อมูลของเข้า |

## หมายเหตุ

- ฝั่งหน้าเว็บ (`index.html`) จะแสดงวันที่ต้องเคลียร์ทิ้งตอนกรอก "จำนวนของเข้า" และส่งค่า `inDate` / `expireDate` ไปกับข้อมูลด้วย — ถ้าอยากเก็บลงชีตเป็นคอลัมน์ ให้เพิ่มคอลัมน์ "วันที่ต้องทิ้ง" ในฝั่ง Apps Script ที่บันทึกข้อมูล
- สคริปต์แจ้งเตือนคำนวณวันหมดอายุจากคอลัมน์ "วันที่เวลา" ที่มีอยู่แล้ว จึง**ใช้ได้กับข้อมูลเก่าทันที** ไม่ต้องแก้โครงสร้างชีต
- การเช็คของเข้าใหม่ใช้วิธีดูว่าชีต "จำนวนของเข้า" มีแถวเพิ่มหรือไม่ ทุก 5 นาที — จึง**ไม่ต้องแก้โค้ดบันทึกข้อมูลเดิมเลย** และการเช็คเฉย ๆ ไม่กินโควต้าข้อความ LINE
- LINE Messaging API แบบฟรีส่งข้อความ push ได้ 200 ข้อความ/เดือน — แจ้งของครบกำหนดวันละ 1 ครั้ง (~30/เดือน) + แจ้งของเข้า (ครั้งละ 1 ข้อความต่อการกรอก 1 รอบ) รวมแล้วปกติไม่เกิน 100 ข้อความ/เดือน ยังเหลือเฟือ
